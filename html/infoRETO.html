General

¿Porqué se llama Proyecto RETO?

Es la contracción de REcursive TOpology.

La programación para tratar las 4+1 topologías (Conjunto, Lista, Arbol y Grafo + Malla) es de tipo recursivo.

El lenguaje empleado es ECMAScript 2015 (ES6), tanto en lado server (Node.js) como en lado cliente.

En lado server se usan algunos scripts tipo CGI, escritos en Bash, Python, PHP, etc etc

¿Cuando y como surgió el proyecto?

Fué al jubilarme en abril del 2010. Entonces se me ocurrió la idea por primera vez, y me pareció un bonito proyecto, al que dedicarle horas para hacerlo viable.

El punto de partida es la idea de que todas las aplicaciones hacen lo mismo.

Almacenar información, y recuperarla, mostrándola mediante un interfaz apropiado, al usuario.

Tecnologías base (stack)

Hay un stack que denomino v-NEMO.
  • Vue.js
  • Node
  • Express
  • MongoDB
  • Oracle

Vue.js renderiza el HTML5, con los datos de la aplicación.

Node.js / Express: sirven tanto los ficheros estáticos, como los datos, via Ajax.

MongoDB / mongoose : Establecen el mecanismo para grabar, actualizar y borrar información persistente.

Oracle es el sinónimo para varias BBDD SQL, accesibles con scripts CGI.

Además, hay muchos packages NPM para ampliar las funcionalidades de Node

Fundamentos

Kernel

El Kernel es el conjunto de librerías JS que son comunes a todas las aplicaciones.

Las librerías son, básicamente Clases y funciones ECMAScript 2015 (ES6).

Son unas 4000 líneas de código (en bruto).El peso total es de 60K, aprox.

Cada librería está dedicada a un aspecto concreto de las funcionalidades.

Las librerias que componen el Kernel v.1 son:

  • libK1_Ajax
  • libK1_Clases
  • libK1_Idioma
  • libK1_Random
  • libK1_Sesion
  • libK1_Tiempo
  • libK1_Topol
  • libK1_Trazo
  • libK1_Utils

Topologías

La información de las aplicaciones se agrupa en colecciones de Nodos

Las colecciones son, básicamente Clases ECMAScript 2015 (ES6), que constan de un Array de Nodos, y los métodos para manipular la topología.

Cada topología lleva asociada un objeto meta para agilizar las búsquedas en MongoDB

Clase rMeta = {tag:'XYZ',iam: iamClaseTopol,org:Organiz}

Se consideran 4 topologías base y una derivada:

  • Conjunto
  • Lista
  • Arbol
  • Grafo
  • --------------------------------------
  • Malla

Nodos

Los nodos representan las unidades de información de la aplicación que se desarrolla.

Partiendo de la clase Nodo, se pueden definir clases que representan las entidades de la aplicación

Comparativamente, si una topología puede asimilarse (relativamente) a una Tabla SQL, los nodos representarían las tuplas/filas

Pero además, estas clases proveen los métodos para manipular las entidades, e incluso la info para validar los formularios de edición, (con la capacidad de soportar MultiLenguaje).

Arcos

Para implementar la topología de Grafo, se define la clase Arco, que sirve para conectar dos Nodos.

Tecnología servidor

Node.js

Hay un prototipo de servidor, con su Model, Routers y Controllers

El Model es siempre el mismo: 

var topolSchema = new Schema({ 
meta : { type: {} },
nodos: { type: [] },
});

Esta definición genérica permite que mongoose maneje por igual cualquier modelo de datos que nos sea útil.

MongoDB

Mediante el package Mongoose se establece el vínculo con las BBDD No-SQL de la aplicación.

BBDD SQL

A través de scripts estilo CGI, se establece comunicación con BBDD SQL : Oracle, MySQL, SQLite, PostGIS, etc
